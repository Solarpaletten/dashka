# 🚀 SOLAR DEV SYSTEM v1.0

**AI-driven development protocol (Dashka | Claude | DeepSeek | Leanid)**

---

## 🎯 ЦЕЛЬ

Обеспечить:

* полный контроль изменений кода
* прозрачность работы AI
* отсутствие “слепых замен файлов”
* быстрый и безопасный деплой

---

## 🧠 ОСНОВНОЙ ПРИНЦИП

```text
КОД НЕ ВАЖЕН → ВАЖНЫ ИЗМЕНЕНИЯ КОДА
```

```text
Не "что написано"  
А "что изменилось и почему"
```

---

## 🧩 РОЛИ В СИСТЕМЕ

### 🔵 Leanid (Architect)

* принимает финальные решения
* утверждает изменения
* контролирует архитектуру

---

### 🟣 Dashka (Coordinator / Senior)

* принимает результат от Claude
* формирует задачи
* направляет DeepSeek на проверку
* готовит diff для Leanid

---

### 🟢 Claude (Engineer)

* пишет код
* реализует фичи
* предлагает изменения

---

### 🟡 DeepSeek (Junior / Reviewer)

* сравнивает файлы
* делает `diff`
* выявляет:

  * лишние изменения
  * риски
  * несоответствия

---

## 🔄 PIPELINE (ОСНОВА)

```text
Claude → Dashka → DeepSeek → Leanid → Git → Deploy
```

---

## ⚙️ ПРОЦЕСС РАБОТЫ

### 1. Генерация

Claude генерирует код / изменение

---

### 2. Передача

Dashka принимает и формирует задачу

---

### 3. Проверка (КЛЮЧЕВОЙ ЭТАП)

DeepSeek выполняет:

```bash
diff -u file_old file_new
```

---

### 4. Анализ

Leanid видит:

```diff
- старая строка
+ новая строка
```

И принимает решение:

```text
✔ ACCEPT
❌ REJECT
✏️ MODIFY
```

---

### 5. Применение

```bash
git add .
git commit -m "clear meaningful message"
git push origin main
```
